# 安全策略

## 报告漏洞

请不要通过公开 Issue 披露未修复的安全问题。请使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告功能，并包含：

- 受影响版本或提交；
- 可复现步骤；
- 可能影响；
- 建议修复方式（如有）。

请勿在报告中提交真实访问 Token、Prompt、会话文件或个人数据库。必要时使用已经撤销的测试凭据和最小化样例。

## 数据边界

Token 会话内容默认仅在本机读取和汇总。启用自托管同步后，客户端只上传日期、设备标识、来源、provider、模型和分类 Token 数；明文访问 Token 保存在 macOS Keychain，服务端仅保存 SHA-256 哈希。
